# 设计模式深度剖析：Go 语言实现与设计原则

## 一、创建型模式

### 1.1 单例模式

```go
// 线程安全的单例实现
type singleton struct {
    value string
}

var (
    instance *singleton
    once     sync.Once
)

func GetInstance() *singleton {
    once.Do(func() {
        instance = &singleton{value: "initialized"}
    })
    return instance
}
```

### 1.2 工厂模式

```go
// 工厂方法模式
type Product interface {
    Use() string
}

type ProductA struct{}
func (p *ProductA) Use() string { return "Product A" }

type ProductB struct{}
func (p *ProductB) Use() string { return "Product B" }

type Factory interface {
    CreateProduct() Product
}

type FactoryA struct{}
func (f *FactoryA) CreateProduct() Product { return &ProductA{} }

type FactoryB struct{}
func (f *FactoryB) CreateProduct() Product { return &ProductB{} }
```

## 二、结构型模式

### 2.1 适配器模式

```go
// 适配器模式
type Target interface {
    Request() string
}

type Adaptee struct{}
func (a *Adaptee) SpecificRequest() string {
    return "adaptee specific request"
}

type Adapter struct {
    adaptee *Adaptee
}

func (a *Adapter) Request() string {
    return a.adaptee.SpecificRequest()
}
```

### 2.2 装饰器模式

```go
// 装饰器模式（Go 中常用中间件模式）
type Handler func(ctx context.Context) error

type Middleware func(Handler) Handler

func Logging(next Handler) Handler {
    return func(ctx context.Context) error {
        log.Println("before")
        err := next(ctx)
        log.Println("after")
        return err
    }
}

func Recovery(next Handler) Handler {
    return func(ctx context.Context) error {
        defer func() {
            if r := recover(); r != nil {
                log.Println("recovered:", r)
            }
        }()
        return next(ctx)
    }
}
```

## 三、行为型模式

### 3.1 观察者模式

```go
// 观察者模式
type Observer interface {
    Update(message string)
}

type Subject struct {
    observers sync.Map
}

func (s *Subject) Attach(o Observer) {
    s.observers.Store(o, struct{}{})
}

func (s *Subject) NotifyAll(message string) {
    s.observers.Range(func(key, _ interface{}) bool {
        key.(Observer).Update(message)
        return true
    })
}
```

### 3.2 策略模式

```go
// 策略模式
type Strategy interface {
    Execute(a, b int) int
}

type Add struct{}
func (Add) Execute(a, b int) int { return a + b }

type Multiply struct{}
func (Multiply) Execute(a, b int) int { return a * b }

type Calculator struct {
    strategy Strategy
}

func (c *Calculator) SetStrategy(s Strategy) {
    c.strategy = s
}

func (c *Calculator) Calculate(a, b int) int {
    return c.strategy.Execute(a, b)
}
```

## 四、Go 语言特有模式

### 4.1 函数选项模式

```go
type Server struct {
    addr    string
    port    int
    timeout time.Duration
    tls     *tls.Config
}

type Option func(*Server)

func WithTimeout(t time.Duration) Option {
    return func(s *Server) { s.timeout = t }
}

func WithTLS(cfg *tls.Config) Option {
    return func(s *Server) { s.tls = cfg }
}

func NewServer(addr string, port int, opts ...Option) *Server {
    s := &Server{
        addr:    addr,
        port:    port,
        timeout: 30 * time.Second,
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}
```

> **总结**：设计模式是解决特定问题的通用方案。单例模式通过 sync.Once 实现，工厂模式通过接口实现，装饰器模式通过函数组合实现，观察者模式通过 sync.Map 实现。在 Go 中，函数选项模式是特有的扩展方式，提供了灵活的 API 设计。
