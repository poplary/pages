## 一、进程调度

### 1.1 CFS（完全公平调度器）

```c
// Linux CFS 调度器

// 核心数据结构
// 每个进程有一个 vruntime（虚拟运行时间）
// 红黑树按 vruntime 排序
// 选择 vruntime 最小的进程运行

// vruntime 计算
// vruntime += 实际运行时间 × (nice_0_weight / 当前进程权重)

// 调度周期
// CPU 时间在可运行进程间分配
// 调度周期 = 6ms × 进程数（但不超过 48ms）
// 每次调度选择一个进程，运行一个时间片
```

### 1.2 进程状态

```c
// Linux 进程状态
// TASK_RUNNING：可运行状态
// TASK_INTERRUPTIBLE：可中断睡眠状态
// TASK_UNINTERRUPTIBLE：不可中断睡眠状态（等待 I/O）
// __TASK_STOPPED：停止状态
// __TASK_TRACED：被跟踪状态
// EXIT_DEAD：退出状态
// EXIT_ZOMBIE：僵尸状态

// 状态转换
// fork() → TASK_RUNNING
// 调度 → 运行
// wait_event() → TASK_INTERRUPTIBLE
// I/O 完成 → TASK_RUNNING
// exit() → EXIT_ZOMBIE
// wait() → 释放
```

## 四、I/O 模型

### 4.1 五种 I/O 模型

```c
// 1. 阻塞 I/O（BIO）
// 进程等待 I/O 完成

// 2. 非阻塞 I/O（NIO）
// 立即返回 EAGAIN，进程轮询

// 3. I/O 多路复用
// select/poll/epoll
// 同时监视多个文件描述符

// 4. 信号驱动 I/O（SIGIO）
// I/O 就绪时发送信号

// 5. 异步 I/O（AIO）
// 内核完成 I/O 后通知进程
```

### 4.2 Epoll 实现

```c
// epoll 是 Linux 的 I/O 事件通知机制
// 三种操作
// 1. epoll_create：创建 epoll 实例
// 2. epoll_ctl：注册/修改/删除事件
// 3. epoll_wait：等待事件

// 事件触发模式
// 1. 水平触发（LT，默认）：只要有数据就通知
// 2. 边缘触发（ET）：只有状态变化时通知

// 边缘触发优势
// 1. 减少 epoll 调用次数
// 2. 需要非阻塞 I/O
// 3. 需要一次性读取所有数据
```

## 五、系统调用

### 5.1 系统调用流程

```c
// 系统调用流程
// 用户态 → 中断/异常 → 内核态 → 执行 → 返回用户态

// 例如 read() 系统调用
// 1. 用户程序调用 read()
// 2. libc 封装，设置系统调用号
// 3. 执行 syscall 指令
// 4. 切换到内核栈
// 5. 执行 sys_read()
// 6. 返回用户态
```

### 5.2 上下文切换

```c
// 上下文切换开销
// 1. 保存 CPU 寄存器
// 2. 切换页表（TLB 刷新）
// 3. 切换内核栈
// 4. 调度器选择下一个进程
// 5. 恢复 CPU 寄存器
// 6. 返回用户态

// 上下文切换优化
// 1. 减少不必要的系统调用
// 2. 使用线程代替进程
// 3. 使用 epoll 代替多线程
// 4. 使用异步 I/O
```

---

> **总结**：Linux 内核通过 CFS 调度器实现公平的进程调度，通过虚拟内存和页面回收机制管理内存，通过 VFS 抽象层支持多种文件系统，通过 epoll 实现高效的 I/O 多路复用。理解这些内核机制是排查系统性能问题和优化应用程序的关键。

## 面试题精选

**Q1: 硬链接和软链接的区别？**

A: 硬链接共享 inode，不能跨文件系统，不能链接目录；软链接有独立 inode，可跨文件系统，可链接目录，原文件删除后链接失效。

**Q2: epoll 的 LT 和 ET 模式的区别？**

A: LT（水平触发）：只要有数据就通知，不需要一次读完；ET（边缘触发）：只有状态变化时通知，必须一次读完，需要非阻塞 I/O。

**Q3: 如何排查 CPU 占用过高？**

A: `top` 找到高 CPU 进程 PID → `top -H -p PID` 查看线程 → `strace -p PID` 跟踪系统调用 → 分析应用日志或代码。

**Q4: grep、sed、awk 的区别？**

A: grep 用于文本搜索；sed 用于流编辑（替换/插入/删除）；awk 用于文本分析和格式化（按列处理）。
